Code Quality

Code quality is loosely coupled

Loose coupling means the code is more independent and easier to test in smaller units.

 Code quality is cohesive

Cohesion means that the code is more focused to deal with only one thing at a time, and it makes code much clearer and more straightforward to work with.

 Code quality is non-redundant

Code that is non-redundant costs less to maintain and extend.

The 5 Service Dimensions

Reliability #1 Just do it

Do what you say you’re going to do when you said you were going to do it.

 Responsiveness #2 Do it now

Respond quickly, promptly, rapidly, immediately, instantly.

 Assurance #3 Know what you are doing

Service providers are expected to be the experts of the service they’re delivering. It’s a given

 Empathy #4 Care about customers

Services can be performed completely to specifications. Yet customers may not feel provider employees care about them during delivery. And this hurts customers’ assessments of providers’ service quality.

 Tangibles #5 Look sharp

Even though this is the least important dimension, appearance matters. Just not as much as the other dimensions.
